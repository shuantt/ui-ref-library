## 預覽

每個風格都是一個獨立的 landing page，展示該設計風格的核心元素、配色、排版與互動效果。

首頁每張卡片提供「風格說明」與「AI Prompt」兩個入口：前者說明風格／產品模式的核心特色與適用情境，後者可直接複製並替換產品、任務與技術棧。功能型卡片會明確標示為 Landing、Data & System、互動模式或設計原則，不會被誤認成純視覺風格。

本庫記錄的是持續擴充的個人設計偏好與示意，不是 UI 風格的完整清單，也不是 Foundry OS 的風格白名單。Foundry OS 在提出客戶 Proposal 時，仍應同時考慮本庫以外的設計知識、專案參考與合理的新方向。

## 技術特點

- **純靜態 HTML** - 無需建置，直接部署
- **Tailwind CSS CDN** - 使用 CDN 引入，無需 npm install
- **Google Fonts** - 各風格搭配適合的字體
- **響應式設計** - 57 個樣板共用手機版安全基線，並保留各自視覺語彙
- **CSS 動畫** - 豐富的互動效果
- **AI 溝通輔助** - 57 筆風格說明、生成 Prompt 與剪貼簿回饋
- **完整性檢查** - 可驗證 57 個索引連結、頁碼與返回路徑

## 快速開始

### 本地預覽

```bash
# 複製專案
git clone https://github.com/shuantt/ui-ref-library.git

# 進入資料夾
cd ui-ref-library

# 用瀏覽器開啟 index.html
open index.html
# 或
xdg-open index.html  # Linux
start index.html     # Windows
```

## 專案結構

```
ui-ref-library/
├── index.html              # 首頁索引
├── README.md               # 說明文件
├── scripts/
│   ├── check-library.mjs   # 內容與連結完整性檢查
│   ├── check-rwd.mjs       # RWD、首頁卡片互動與可選視覺截圖
│   └── style-card-guides.js # 57 筆風格說明、Prompt 與對話框互動
└── styles/
    ├── responsive.css      # 全樣板共用的行動版安全基線
    ├── 01-minimalism.html
    ├── 02-neumorphism.html
    ├── 03-glassmorphism.html
    ├── ...
    └── 57-gen-z-chaos.html
```

- **產品經理** - 與團隊溝通設計方向時的參考
- **學習者** - UI/UX 設計入門的實例教材

## 自動檢查

```bash
# 連結、標題、頁碼與基本結構
node scripts/check-library.mjs

# 全部 58 個頁面的手機版水平溢位，以及首頁 57 張卡片／114 個操作
node scripts/check-rwd.mjs --width=375

# 桌面版回歸
node scripts/check-rwd.mjs --width=1280
```

RWD 檢查需要本機已安裝 Chrome 或 Edge。需要留下指定頁面的完整截圖時，可加上
`--capture=47-exaggerated-minimalism.html --capture-dir=./rwd-captures`。
